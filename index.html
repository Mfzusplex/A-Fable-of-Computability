Author: Lucian Wilber

Module I: Semiblocks

Introduction

Semiblocks is a non-executable pseudocode language that extends the traditional binary code framework and allows the embedding of C++ code within its functions. It is intended for theoretical modeling, black-box analysis, hierarchical process representation, and formal exploration of computation.

The language is processed sequentially from left to right. Functions defined outside of blocks have greater privilege in terms of scope, whereas blocks provide localized structure for computations.

Core Notation

Atoms of the code: A = 1, B = 0

Syntactic atoms: / = SPACE, # = TAB, ## = ENTER

Commentary: :: serves as a prefix for comments (e.g., :: HELLO WORLD)

Functions and blocks: { = FUNC START (suffix of function name), } = FUNC END, [[ = BLOCK START (suffix of block name), ]] = BLOCK END

Conditions: CS/# = CONDITION START, #/CE = CONDITION END. Logic gates are admitted within conditions.

Extra instructions: DOWN() decrement value, UP() increment value, RET = Return value, INT = Define integer.

Embedded C++ code is permitted within functions for computation and modeling purposes.

Example: Fast-Growing Hierarchy Extended to Uncountable Ordinals

:: Fast-Growing Hierarchy Extended to Uncountable Ordinals
{FGH##

INT alpha/    :: Current ordinal index
INT n/        :: Level in hierarchy

[[base_case##
    CS/# alpha==0/
        RET 1/
    #/CE
]]

[[successor_case##
    CS/# alpha>0 AND is_successor(alpha)/
        :: Use previous level
        RET FGH(alpha-1, n+1)/
    #/CE
]]

[[limit_case##
    CS/# is_limit(alpha)/
        :: Embed fundamental sequence: alpha[n]
        RET FGH(fund_seq(alpha, n), n)/
    #/CE
]]

[[uncountable_extension##
    CS/# is_uncountable(alpha)/
        :: Symbolic extension using ℵ₁, ℵ₂, ...
        :: We cannot compute exact value, but we can define notation
        RET FGH(symbolic(alpha), n)/
    #/CE
]]
}

Module II: Beyond Uncomputability

Introduction

Module II explores the boundaries of computability within formal pseudocode, using higher-order ordinal functions and black-box oracles. The goal is to define operations and hierarchies that extend beyond conventional algorithmic reach, addressing ordinals that cannot be effectively computed by machines with finite instruction sets or access to standard recursive operations.

Pseudocode Description

:: Function alpha computes fundamental sequences of ordinals
function alpha(n: Nat, m: OrdNotation) -> OrdNotation:
    if n == 0 and m == 0:
        return 0
    else:
        return fund_seq(m, n)    // m[n]

:: Collapse function for ordinal sequences
function collapse(k: OrdNotation, n: Nat) -> OrdNotation:
    return fund_seq(k, n)       // total operation as described above

:: Beta function combines collapse with fast-growing hierarchy evaluation
function beta(n: Nat, k: OrdNotation) -> Nat:
    if n == 0 and k == 0:
        return 0
    else:
        i := collapse(k, n)    // i = k[n]
        return f_i(n)          // evaluate FGH at index i

:: Tau defines the least ordinal uncomputable by n-th order black-box oracle machines
function tau(n: Nat) -> OrdNotation:
    return sup( Ord_beta^(n) )   // least ordinal not computable by such machines

:: Gamma defines the least ordinal uncomputable by n-th order axiom oracle machines
function gamma(n: Nat) -> OrdNotation:
    return sup( Ord_tau^(n) )    // least ordinal not computable by such machines

Discussion

The functions alpha, collapse, and beta provide a framework for systematically traversing and evaluating ordinals using fundamental sequences and hierarchical computation. The tau and gamma functions formalize the notion of ordinals that exceed the computational power of n-th order oracle machines, thus exploring a formal notion of “beyond computability.”

By combining fundamental sequences, collapsing operations, and hierarchical evaluation, this module situates itself at the interface of ordinal analysis, theoretical computer science, and the philosophy of uncomputability.
